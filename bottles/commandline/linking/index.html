Linked Project Bottles

bottles linkLinks a package folder to an application folder in development mode

This command will write to a file named .links underneath the main application directory specified by the AppFolder input.
Usages
List the current links for the applicationbottles link <appfolder>
Create a new link for the application to the packagebottles link <appfolder> <bottlefolder> [-r, --remote]
Remove any existing link for the application to the packagebottles link <appfolder> <bottlefolder> [-r, --remove]
Remove any and all existing links from the application to any package folderbottles link <appfolder> [-c, --clean-all]

Arguments
appfolderThe physical folder (or valid alias) of the main application
bottlefolderThe physical folder (or valid alias) of a bottle

Flags
[-r, --remove]Remove the package folder link from the application
[-c, --clean-all]Remove all links from an application manifest file
[-r, --remote]Create a 'remote' link that will run the service or app at the BottleFolder in a separate AppDomain
[-b, --bootstrapper <bootstrapper>]Explicitly specify the name of the Bottles bootstrapper class for a remote link
[-f, --config-file <configfile>]Specify the name of the configuration file to use for a remote link

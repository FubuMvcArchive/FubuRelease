Adding Projects to Existing Solutions

fubu addAdds new projects to an existing solution

The fubu add command creates new projects to an existing solution. Run this command from the root directory of your code repository. At this point, all the project types for add are the same as fubu new.
Usages
defaultfubu add <projectname> <profile> [-s, --solution <solution>] [-s, --short-name <shortname>] [-n, --no-tests] [-l, --list] [-o, --options [<options1 options2 options3 ...>]] [-d, --dot-net <dotnet>]
listfubu add [-l, --list]
Arguments
projectnameThe name of the new project
profileProject profile. Use the --list flag to see the valid options
Flags
[-s, --solution <solution>]Specify the solution file name if there is more than one in this code tree
[-s, --short-name <shortname>]Used in many templates as a prefix for generted classes
[-n, --no-tests]Do not generate a matching testing project. Boo!
[-l, --list]List all the possible project types and their valid options
[-o, --options [<options1 options2 options3 ...>]]Extra options for the new application
[-d, --dot-net <dotnet>]Specify the .Net version 'v4.0' or 'v4.5'

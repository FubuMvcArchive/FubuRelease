New Solutions and Code Trees

In all usages, the fubu new command will generate an instructions.txt file that will explain the items and files generated during templates.

Empty Solutions

If none of the project templates match what you want or you only want to generate a new solution without any projects, just use this:

fubu new MySolution empty

FubuMVC Applications

The default for fubu new is a brand new FubuMVC application, so the basic usage is just:

fubu new MyApp

The default choices will build out a new FubuMVC application that uses

See Using as a Development Server for more information.

The rake tasks for your newly generated FubuMVC solution should be something like this:

rake ci                # Target used for the CI server
rake clean             # Prepares the working directory for a new build
rake compile           # Compiles the solution src/MyApp.sln
rake default           # **Default**, compiles and runs tests
rake docs:bottle       # 'Bottles' up a single project in the solution with...
rake docs:run          # Runs a documentation project hosted in FubuWorld
rake docs:run_chrome   # Runs the documentation projects in this solution i...
rake docs:run_firefox  # Runs the documentation projects in this solution i...
rake docs:snippets     # Gathers up code snippets from the solution into th...
rake myapp:alias       # Add the alias for src/MyApp
rake myapp:chrome      # run the application with Katana hosting and 'watch...
rake myapp:firefox     # run the application with Katana hosting and 'watch...
rake myapp:restart     # touch the web.config file to force ASP.Net hosting...
rake myapp:run         # run the application with Katana hosting
rake ripple:history    # creates a history file for nuget dependencies
rake ripple:package    # packages the nuget files from the nuspec files in ...
rake ripple:restore    # Restores nuget package files and updates all float...
rake ripple:update     # Updates nuget package files to the latest
rake sln               # Open solution src/MyApp.sln
rake unit_test         # Runs unit tests for MyApp.Testing
rake version           # Update the version information for the build

When you look in the code for the new FubuMVC application, you'll see these things: * An IApplicationSource class that specifies how your application is bootstrapped * A FubuRegistry class * A StructureMap Registry class * A class named HomeEndpoint that represents the root url or home page of your new application * Global.asax.cs to bootstrap FubuMVC for ASP.Net hosting

At this point, you should be able to simply press F5 and run the application with the home page coming up as text with instructions about how to get started with the code in your new application.

FubuMVC Application with the Spark View Engine

Using the Spark view engine in the generation is done with the --options flag:

fubu new MyApp --options spark

This option will add several extra Nuget's for html conventions, using assets, and the Spark view engine support. The templating will also add a default project layout and some Spark bindings for common FubuMVC view helpers.

FubuMVC Application with the Razor View Engine

You can also generate a new FubuMVC application using the Razor view engine with this option usage:

fubu new MyApp --options razor

FubuMVC Bottles

You can generate a new solution for an FubuMVC Bottle with this usage:

fubu new MyFeatureBottle web-bottle

This generation will add: * A FubuPackageRegistry -- TODO(link to the FubuMVC topic for this) * An implementation of IFubuRegistryExtension for global policies * Adds the [FubuModule] attribute to the assembly properties marking it as a Bottle

Simple Class Libraries

To generate a simple class library project, use:

fubu new MyLibrary library

The templating will generate a new project with a reference to FubuCore, the idiomatic CommonAssemblyInfo.cs linked file, and a testing library.

FubuTransportation Service

You generate a brand new FubuTransportation service with this usage:

fubu new MyService transport-service

This will generate a new project and test library that has a reference to FubuTransportation, FubuTransportation.LightningQueues, and BottleServiceRunner TODO(add the link!). You will also get rake tasks to install, uninstall, stop, and start the service as well as tasks to run the service in console mode in either TopShelf hosting or the fubu run command.

We will be adding a new template for generating a FubuTransportation bottle soon

Bottle Service

If you wish to create a new Windows service hosted with TopShelf that is not using FubuTransportation, use the bottle-service template shown below:

fubu new MyService bottle-service

Arguments
solutionnameName of the solution and the root folder without an extension
profileProject profile. Use the --list flag to see the valid options

Flags
[-l, --list]List all the possible project types and their valid options
[-p, --preview]Only list a preview of the template plan, but do not execute the plan
[-e, --edge]Use edge nugets from the Fubu TeamCity feed. Equivalent to --ripple Edge
[-f, --floating]Use floating edge nugets from the Fubu TeamCity feed. Equivalent to --ripple FloatingEdge
[-r, --ripple FloatingEdge|Edge|PublicOnly]Choose a ripple configuration for only public Nuget feeds, including the Fubu TeamCity feed, or 'floating' on the Fubu edge
[-s, --short-name <shortname>]Used in many templates as a prefix for generated classes
[-c, --clean]Clean out any existing contents of the target folder before running the templates
[-n, --no-tests]Do not generate a matching testing project. Boo!
[-v, --version <version>]Visual Studio version. 'VS2012' is the default. Options are 'VS2010', 'VS2012', or 'VS2013'
[-d, --dot-net <dotnet>]Specify the .Net version 'v4.0' or 'v4.5'
[-i, --ignore]Ignore the presence of existing files
[-o, --options [<options1 options2 options3 ...>]]Extra options for the new application

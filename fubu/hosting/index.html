Using as a Development Server

The fubu run command allows you to run FubuMVC or FubuTransportation applications hosted with Katana. Even though you're most likely hosting your FubuMVC web applications on IIS, nginx, or Apache, the Katana based development server described here can provide a more efficient development experience.

How do I set it up?

The fubu run hosting needs just two things:

  1. The root directory of your FubuMVC or FubuTransportation application project
  2. An IApplicationSource class to use to bootstrap your application (as described in the FubuMVC Bootstrapping docs)

By default, fubu run assumes that you're running an application in the current directory, but you can specify the directory name with this usage:

fubu run --directory src/MyApp

The --directory flag can be either an absolute path or a relative path from the current directory.

fubu run has some ability to automatically determine the IApplicationSource class following these rules:

  1. If there is only a single concrete IApplicationSource class in all of the assemblies in the application, use that one
  2. If there are multiple candidates, look for a single candidate in an assembly whose name matches the containing folder. I.e., if you're running an application in a folder named "c:\code\application\src\MyApp", look for the "MyApp.dll" assembly.
  3. If there are still multiple candidates, the command line will list out all of the possible candidate classes and return without starting the application.

You can specify the type name of the specific IApplicationSource class you want to use with the --application flag like this:

fubu run --application TestingApplication

Also check out the FubuRake helpers for running applications with rake and fubu run.

Recycling the Application and 'Watched' Mode

fubu run hosts the running application in a separate AppDomain using shadow copying. The big advantage to this approach is that you're able to recompile your application even while the fubu hosting is running the application.
Out of the box, the fubu run hosting will tear down the existing AppDomain and create a new one whenever it detects changes to any binary file or the web.config file.

Beyond that, if you use the watched mode with the --watched flag, fubu run will launch a browser with WebDriver so that the hosting can refresh the current web page whenever content changes are detected in the running application. In fullblown watched mode, the fubu hosting is looking for these kinds of changes:

  1. Content changes in views or assets that would cause a browser refresh
  2. Application changes in *.config files that would necessitate restarting the application, but not the running AppDomain
  3. AppDomain changes in binary files that would cause a completely new AppDomain

The file matching logic for watching files (as of Dec. 2013) is:

*.spark=Content
*.dll=AppDomain
*.exe=AppDomain
*.script.config=Application
*.asset.config=Application
*.assets.config=Application
*.cshtml=Content
*.css=Content
*.js=Content
*.coffee=Content
web.config=AppDomain
*.config=Application
A future version of the fubu gem may make these rules configurable.

fubu runRun a fubumvc application hosted in Katana

fubu run[-b, --build ] [-p, --port ] [-a, --application ] [-c, --config ] [-d, --directory ] [-o, --open] [-w, --watched] [-b, --browser Chrome|Firefox] [-p, --production-mode]
Flags
[-b, --build <build>]If you are running a class library, sets the preference for the profile to load. As in bin/[BuildFlag]. Default is debug
[-p, --port <port>]IP Port. Default is 5500
[-a, --application <application>]Specific name of an IApplicationSource class that builds this application
[-c, --config <confi>]If set, overrides the name of the configuration file for the FubuMVC application AppDomain to use
[-d, --directory <directory>]Overrides the directory that is the physical path of the running fubumvc application
[-o, --open]Start the default browser to the home page of this application
[-w, --watched]Open a 'watched' browser with WebDriver to refresh the page on content or application recycles
[-b, --browser Chrome|Firefox]Browser to use in watched. Default is Firefox because it's more stable.
[-p, --production-mode]Unless this flag is set, the fubumvc application will run in Development mode

Project Item Generation

This funtionality is very new and the signature is likely to change in later versions
The thought is that this functionality migrates to a forthcoming fubu generate command with options for: 
fubu generate spark [name]

fubu generate razor [name]

fubu viewCreates and attaches a view model/view pair to the project in the current folder

At the moment, fubu view only generates Spark views. You need to call this command from the exact folder where you want the view file and matching input models and endpoint classes to be written. This command will attach the new *.cs files to the *.csproj file in the root of the current project with the namespace derived from the folder names.
fubu view [-u, --url ] [-o, --open]
Arguments
nameName of the view and matching model without file extension
Flags
[-u, --url <ur>]If specified, will make this actionless view applied to the given url pattern
[-o, --open]open the view model and view after generation

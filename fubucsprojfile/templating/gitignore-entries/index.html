Gitignore Entries

You can direct git to ignore files like build products, test results, and generated code items by using the .gitignore file.

Adding entries to the .gitignore file of a target solution is accomplished by simply adding a file named ignore.txt with the entries that you'd like to have added to your .gitignore.

Each line is added separately to the .gitignore file and duplicates are ignored.

The ignore.txt format is just a text pattern per line:

push.bat
TestResult.xml
build
build35
artifacts
obj
bin
deploy
deploy/*
src/CommonAssemblyInfo.cs
_ReSharper.*
*.csproj.user
*.resharper.user
*.ReSharper.user
*.suo
*.cache
~$*
*.userprefs
*.pidb
test-results
src/packages
*.dotCover
src/*/repositories.config
.bottle-alias
results
*.DotSettings.user
Gemfile.lock
logs
pak*.zip

This sample was taken for the template for a brand new solution in fubu new at https://github.com/DarthFubuMVC/fubu/blob/master/templates/solution/baseline/ignore.txt.

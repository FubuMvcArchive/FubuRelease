Gem Dependencies

Many of the Fubu projects use Ruby Gems and Bundler for dependency management of command line tools, especially tools that are used in build automation scenarios.

To declare new gem dependencies in a template, simply add a file named gems.txt to the root of your template. The format of this file per line is [gem name],[version spec], where the version spec is per the same syntax you would use inside of a Gemfile file.

When the template is executed, the gem declarations will be added to the Gemfile file (the templating will create a new file if one does not already exist) at the root of the code tree.

This sample is taken from the default template that fubu new uses to generate all new code trees with a functioning rake script.

rake,>=10.0.3
bundler,>=1.3.5
fuburake,~>0.5
ripple-cli,~>2.0
fubudocs,~>0.9
At this point we assume that all gems are available from the default Rubygems feed.

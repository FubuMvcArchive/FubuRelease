Template Inputs

Some of the template options like code files allow you to express substitutions in text that will be replaced by a value supplied by the template request.

For an example, look at this sample taken from the fubu templates for a file named %FUBU_REGISTRY%.cs:

using FubuMVC.Core;

namespace %NAMESPACE%
{
    public class %FUBU_REGISTRY% : FubuRegistry
    {
        public %FUBU_REGISTRY%()
        {
            // Register any custom FubuMVC policies, inclusions, or 
            // other FubuMVC configuration here
            // Or leave as is to use the default conventions unchanged
        }
    }
}

When the template containing this file is executed, the text items %NAMESPACE% and %FUBU_REGISTRY% will be replaced in both the resulting text and the file name with the values of those substitutions in the template request.

The usage of '%' and all caps in the substitution names is simply an idiom and not a requirement

Built in Substitutions

FubuCsProjFile has a couple special substitutions whose values are supplied by the templating engine based off the initial TemplateRequest itself:

  1. %SOLUTION_NAME% -- the name of a new solution without a file extension
  2. %SOLUTION_PATH% -- the relative path to the solution file from the root of the code repository
  3. %INSTRUCTIONS% -- the generated instructions to the user describing the template operation
  4. %NAMESPACE% -- only valid inside of code files to a project, the current namespace based on project name and current folder
  5. %ASSEMBLY_NAME% -- only valid inside of code files to a project, the assembly name of the current project
  6. %SHORT_NAME% -- generally meant to be a prefix for other classes. If not specified, FubuCsProjFile determines this by trying to turn "MyCompany.SuperProject" into "SuperProject."
  7. %PROJECT_PATH% -- the relative path from the code repository to the current csproj file
  8. %PROJECT_FOLDER% -- the relative path from the code repository to the folder containing the current csproj file

Defining template specific inputs

You can specify requirements for custom inputs to new templates by adding a file called inputs.txt to a template folder. That file is a flat file that follows this format:

%INPUT_NAME1%=some default value,some description about what %INPUT_NAME% means
%INPUT_NAME2%=some default value,some description about what %INPUT_NAME% means
%INPUT_NAME3%=some default value,some description about what %INPUT_NAME% means

Default values can be built using other substitutions like this example from the FubuMVC project generation:

%FUBU_REGISTRY%=%SHORT_NAME%FubuRegistry,The name of the FubuRegistry class for the application

You can see an example of using inputs.txt in the empty FubuMVC template from fubu.exe.

Code Files

The templating engine will treat each file with the .cs extension as a code file that will be attached to the current csproj file.

Substitutions are processed against code files in both the name of the file and the contents of the file. Code files can be at the root namespace of the project or nested within child directories. The built in %NAMESPACE% substitution uses any child folder nesting to determine the namespace.

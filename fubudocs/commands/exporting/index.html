Exporting to Static Html

fubudocs exportExports static html content for all of the documentation projects in the specified folder

Exporting can be done in three modes,

  1. GhPagesRooted - Use this option if the static content will be hosted at the root of a web application. Every file is index.html as GitHub pages expects, but all links are formed as absolute url's in the exported docs.
  2. GhPagesChildFolder - Use this option if the static content will be hosted in a child folder of a web application (i.e. http://darthfubumvc.github.io/fubudocs). Again, every file is exported as index.html, but the links are relative links.
  3. HtmlDump - This option is for when you just want the documentation content to be browseable from the file system. All topic links include the index.html file and the links are always relative.
fubudocs export [-m, --mode GhPagesRooted|GhPagesChildFolder|HtmlDump] [-i, --include-projects ] [-m, --message ] [-c, --clean] [-r, --root] [-n, --no-bottling] [-w, --watched] [-b, --browser Chrome|Firefox] [-b, --build ] [-o, --open] [-h, --host ]
Arguments
outputThe directory to output the exported content
Flags
[-m, --mode GhPagesRooted|GhPagesChildFolder|HtmlDump]Change the way url's are treated in the exported html content
[-i, --include-projects <includeprojects>]Comma separate list of the projects to include in the export (e.g., fubudocs, myproject)
[-m, --message <message>]Specify the git commit message if exporting directly to gh-pages
[-c, --clean]Delete any existing content in the export directory first
[-r, --root]If selected, all url's in the exported pages will be absolute url's
[-n, --no-bottling]Disables the bottle and code snippet scanning while this command runs
[-w, --watched]Open a 'watched' browser with WebDriver to refresh the page on content or application recycles
[-b, --browser Chrome|Firefox]Browser to use in watched. Default is Firefox because it's more stable.
[-b, --build <build>]If you are running a class library, sets the preference for the profile to load. As in bin/[BuildFlag]. Default is debug
[-o, --open]Start the default browser to the home page of this application
[-h, --host <host>]Specify the directory for a hosting application for your documentation. Will be loaded as a FubuMVC Bottle

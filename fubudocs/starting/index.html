Starting

Starting a new Document Project

The first step is to simply create a new Visual Studio class library project named with the suffix .Docs like FubuCore.Docs or FubuMVC.Docs. You do not need any other references or nugets to get started, but you can add other dependencies as needed for authoring examples if you wish.

Assuming that you're using FubuRake for the build script, all you need to do is to follow the instructions at Enabling FubuDocs with FubuRake. Otherwise, see the command line reference at Command Line Documentation to use fubudocs.exe directly.

You can immediately launch your new FubuDocs project in a FubuMVC hosted website by either running fubudocs run from the command line at the root of your source repository or by using the rake docs:run Rake task. FubuDocs will typically run your documentation website at http://localhost:5050, but the actual port number can vary if there's a conflict.

There's no need to do anything else, but do note that the first time you run the documentation project, the fubudocs run command will call the fubudocs bottle command first to seed the documentation project with the project.xml file described in the next section and the "table of contents" page that you can alter later.

project.xml

FubuWorld looks for a file named "project.xml" in the root of each documentation project for data that is common to all documentation projects. This file contains metadata about the documentation project that FubuDocs uses to generate navigation elements and our standard project summary information.

Again using FubuCore as an example, that file would look like the following:

<?xml version="1.0"?>
<ProjectRoot xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
  <Name>FubuCore</Name>
  <GitHubPage>http://github.com/DarthFubuMVC/FubuCore</GitHubPage>
  <UserGroupUrl>https://groups.google.com/forum/?fromgroups#!forum/fubumvc-devel</UserGroupUrl>
  <BuildServerUrl>http://build.fubu-project.org/project.html?projectId=project6&amp;tab=projectOverview</BuildServerUrl>
  <BottleName>FubuCore.Docs</BottleName>
  <Url>fubucore</Url>
</ProjectRoot>

The elements of note are:

  • Name - the user facing name for this content
  • Tagline - short description of the documentation project
  • Description - longer description of the documentation project
  • GitHubPage - link to the code repository that is covered by this documentation project
  • UserGroupUrl - link to the user group for questions about this content
  • BuildServerUrl - link to the specific project page on TeamCity
  • BottleName - just the name of this bottle. Typically just the assembly name.
  • Url - the root url for all topics in this documentation project
  • NugetWhiteList - an optional comma delimited list of the nuget's published by this code repository
  • PluginTo - optional parent name of another documentation project for relationships like FubuMVC.CodeSnippets to FubuMVC
  • TeamCityBuildType - the TeamCity build id. Should look like "bt93"
  • TwitterHandle - optional twitter account for the project

Do note that the rake docs:bottle task can make some reasonable guesses about many of these values based on the documentation project name.
